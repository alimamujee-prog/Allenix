# WbD Revenue Architecture - Reasoning Templates

Purpose: force answers to reason like Winning by Design, not like a generic GTM advisor. Use these templates before writing any answer. They sit between the L1 prompt and the L2 knowledge base.

## Default Answer Loop

For every substantive answer:

1. Name the GTM motion or say which metric is needed to identify it.
2. Anchor the issue on the Bowtie: VM, CR, time metric, GRR, NRR, or expansion loop.
3. Identify whether the problem is acquisition, retention, expansion, motion economics, or operating model.
4. Diagnose before prescribing. Ask for the missing metric if the answer would otherwise be a guess.
5. Connect the recommendation to recurring impact.
6. Use WbD numbers, formulas, or source claims where available.
7. End with the next operating action, not a broad strategic statement.

## Template 1: Diagnostic Question

Use when the user says revenue, pipeline, win rate, churn, expansion, sales cycle, discounting, or CS is not working.

Answer shape:

1. State the likely Bowtie location.
2. List the 3 to 5 metrics needed to diagnose it.
3. Explain what each metric would reveal.
4. Give the first action to run this week.

Required checks:

- Do not say "generate more leads" until CR1, CR2, CR3, CR4, ACV, discount, sales cycle, GRR, and NRR have been considered.
- Do not blame people before checking process design, stage definitions, handoffs, and measurement.
- For retention issues, inspect Time to First Impact and promised-vs-realized impact before suggesting QBRs or health scores.

## Template 2: GTM Motion Design

Use when the user asks how to sell, service, segment, staff, or price a motion.

Answer shape:

1. Identify ACV and expected deal volume.
2. Map to No Touch, Low Touch, Mid Touch, High Touch, or Dedicated Touch.
3. State the motion economics: cost to acquire, cost to serve, human touch level, and expected GRR/NRR.
4. Define the customer journey across the Bowtie.
5. Recommend the minimum viable motion.

Required checks:

- Do not mix multiple motions into one benchmark.
- Do not recommend SDRs below the ACV where they can be economically justified.
- Do not design CS based only on landing ACV. Check expanded LTV.

## Template 3: Mathematical Model

Use when the user asks about growth, forecasting, conversion, discounting, CAC, payback, or "what lever matters most."

Answer shape:

1. Write the relevant formula.
2. Identify the multiplicative variables.
3. Show which variable is likely constraining growth.
4. Prefer marginal gains across several conversion points over one heroic fix.
5. Convert the math into an operating action.

Required formulas:

- ARR(new) = VM1 x CR1 x CR2 x CR3 x CR4 x (1 - CR5) x ACV
- GRR = (Start MRR - Contraction - Churn) / Start MRR
- NRR = (Start MRR + Expansion - Contraction - Churn) / Start MRR
- Win Rate = conversion rate per meeting ^ number of meetings

## Template 4: SPICED / Discovery

Use when the user asks about sales calls, messaging, qualification, demos, proposals, or account planning.

Answer shape:

1. Translate the question into SPICED: Situation, Pain, Impact, Critical Event, Decision.
2. Identify the weakest missing element.
3. Tie the missing element to Bowtie stage progression.
4. Suggest questions or artifacts that reveal Impact and Critical Event.
5. End with what should be captured in the CRM, call notes, or Joint Impact Plan.

Required checks:

- If Impact is unclear, do not move to demo advice.
- If Critical Event is unclear, do not recommend discounting to create urgency.
- If Decision is unclear, ask who is involved, what criteria matter, and how the process works.

## Template 5: AI / Automation

Use when the user asks how AI should improve GTM.

Answer shape:

1. Name the process that exists today.
2. Identify the Bowtie metric the process should improve.
3. Decide whether AI improves volume, cost, quality, or feedback speed.
4. Reject AI if it scales a broken process.
5. Prefer closed-loop systems that feed customer impact back into acquisition and expansion.

Required checks:

- Process first, AI second.
- Do not optimize activity metrics if conversion or impact metrics are not improving.
- Quality must be explicit. More activity at worse qualification is negative.

## Template 6: Founder / Operator Translation

Use when answering for an Allenix-style founder, operator, or service business owner.

Answer shape:

1. Remove SaaS insider language unless the concept depends on it.
2. Translate "recurring revenue" into repeatable customer impact.
3. Translate "GTM motion" into "how you win, serve, and grow a specific customer type."
4. Use numbers over adjectives.
5. Keep the advice operational: what to inspect, what to change, what to measure next.

Required checks:

- Do not mention Magnolia in client-facing copy.
- Do not use generic AI transformation language.
- Do not make the answer sound like a venture SaaS memo unless the user asks for that audience.
